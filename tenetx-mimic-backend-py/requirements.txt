# --- /verify-metadata route (todo 4) ----------------------------------------
# httpx: server-side fetch of the IdP SAML descriptor (index.ts:169's `fetch`).

# --- /saml/acs form parsing (todo 11: surfaced by the in-container pytest) ---
# POST /saml/acs reads the Keycloak SAML HTTP-POST binding via
# `await request.form()` (app/routes/saml_acs.py:352). The host verification venvs
# carried an OLDER Starlette that parsed urlencoded forms without python-multipart,
# so todo 9 shipped without this pin. The Docker image's floor-pinned resolution
# pulls Starlette 1.3.1, which now ASSERTS python-multipart is installed for ANY
# form parsing (urlencoded included) — without it every real SAML POST 500s on
# `AssertionError: The python-multipart library must be installed`. This is the
# FastAPI-documented form dependency; adding it makes the production image's
# /saml/acs actually work (proven by pytest app/test/test_saml_acs.py inside the
# container). requirements-only change; no app/ source touched.
python-multipart>=0.0.9
